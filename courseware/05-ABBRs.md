# DevSecOps Abbreviations Cheat Sheet

## Security Testing

- **SAST** – Static Application Security Testing  
  Pronunciation: *sast* (like “past”)  
  Brief: Analyzes source code without running the application.

- **DAST** – Dynamic Application Security Testing  
  Pronunciation: *dast* (like “fast”)  
  Brief: Tests a running application by simulating external attacks.

- **IAST** – Interactive Application Security Testing  
  Pronunciation: *eye-ast*  
  Brief: Uses agents inside the app to detect vulnerabilities in real time.

- **SCA** – Software Composition Analysis  
  Pronunciation: *S-C-A*  
  Brief: Scans third-party libraries for known vulnerabilities.

- **RASP** – Runtime Application Self-Protection  
  Pronunciation: *rasp*  
  Brief: Protects applications from attacks during runtime.

## DevOps & Pipeline

- **CI** – Continuous Integration  
  Pronunciation: *C-I*  
  Brief: Automatically builds and tests code on every commit.

- **CD** – Continuous Delivery / Deployment  
  Pronunciation: *C-D*  
  Brief: Automates release or deployment process.

- **CI/CD** – Continuous Integration & Delivery  
  Pronunciation: *C-I C-D*  
  Brief: End-to-end automated pipeline.

  Pronunciation: *O-S*

## Security & Identity

- **IAM** – Identity and Access Management  
  Pronunciation: *I-A-M*  
  Brief: Controls user access and permissions.

- **RBAC** – Role-Based Access Control  
  Pronunciation: *R-back* / *R-B-A-C*  
  Brief: Permissions assigned based on roles.

- **JWT** – JSON Web Token  
  Pronunciation: *J-W-T*  
  Brief: Token used for authentication/authorization.

- **OAuth** – Open Authorization  
  Pronunciation: *oh-auth*  
  Brief: Delegated authorization framework.

- **MFA** – Multi-Factor Authentication  
  Pronunciation: *M-F-A*  
  Brief: Requires multiple verification methods.

## Vulnerabilities & Standards

- **OWASP** – Open Web Application Security Project  
  Pronunciation: *oh-wasp*  
  Brief: Publishes security standards like Top 10 risks.

- **XSS** – Cross-Site Scripting  
  Pronunciation: *X-S-S*  
  Brief: Injecting malicious scripts into web pages.

- **CSRF** – Cross-Site Request Forgery  
  Pronunciation: *C-surf*  
  Brief: Tricks users into performing unintended actions.

- **SSRF** – Server-Side Request Forgery  
  Pronunciation: *S-S-R-F*  
  Brief: Forces server to make malicious internal requests.

- **SQLi** – SQL Injection  
  Pronunciation: *S-Q-L eye*  
  Brief: Injecting SQL queries via input fields.

- **CVE** – Common Vulnerabilities and Exposures  
  Pronunciation: *C-V-E*  
  Brief: Standard identifier for known vulnerabilities.

## Monitoring & Logging

- **SIEM** – Security Information and Event Management

- **SOC** – Security Operations Center  
  Pronunciation: *sock*

- **APM** – Application Performance Monitoring  
  Pronunciation: *A-P-M*

## Containers & Security Tools

- **CSP** – Content Security Policy  
  Pronunciation: *C-S-P*  
  Brief: Prevents XSS by restricting resources.

- **HSTS** – HTTP Strict Transport Security  
  Pronunciation: *H-S-T-S*  
  Brief: Enforces HTTPS usage.

- **TLS** – Transport Layer Security  
  Pronunciation: *T-L-S*  
  Brief: Encryption protocol for secure communication.

- Longer ones → spelled out (JWT, SCA, CVE)
